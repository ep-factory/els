registerValidatorsForColumn ( $  column  )  [protected]

Registers column type specific validators if necessary

We'll first collect the validators/rule infos and add them later on to have them appended to the table tag as a block.

CreoleTypes are:

BOOLEAN BIGINT, SMALLINT, TINYINT, INTEGER FLOAT, DOUBLE, NUMERIC, DECIMAL, REAL BIGINT, SMALLINT, TINYINT, INTEGER TEXT BLOB, CLOB, BINARY, VARBINARY, LONGVARBINARY DATE, YEAR, TIME TIMESTAMP

We will add the following type specific validators:

for notNull columns: required validator for unique indexes: unique validator for varchar types: maxLength validators (CHAR, VARCHAR, LONGVARCHAR) for numeric types: maxValue validators (BIGINT, SMALLINT, TINYINT, INTEGER, FLOAT, DOUBLE, NUMERIC, DECIMAL, REAL) for integer and timestamp types: notMatch validator with [^]+ (BIGINT, SMALLINT, TINYINT, INTEGER, TIMESTAMP) for float types: notMatch validator with [^\.]+ (FLOAT, DOUBLE, NUMERIC, DECIMAL, REAL)

Paramètres:
object $column The Creole column
Renvoie:
void
À faire:

find out how to evaluate the appropriate size and adjust maxValue rule values appropriate

find out if float type column values must always notMatch('[^\.]+'), i.e. digits and point for any db vendor, language etc.
