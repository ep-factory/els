generateCacheKey ( $  internalUri,
$  hostName = '',
$  vary = '',
$  contextualPrefix = '' 
)

Generates a unique cache key for an internal URI. This cache key can be used by any of the cache engines as a unique identifier to a cached resource

Basically, the cache key generated for the following internal URI: module/action?key1=value1&key2=value2 Looks like: /localhost/all/module/action/key1/value1/key2/value2

Paramètres:
string $internalUri The internal unified resource identifier Accepts rules formatted like 'module/action?key1=value1&key2=value2' Does not accept rules starting with a route name, except for ''
string $hostName The host name Optional - defaults to the current host name bu default
string $vary The vary headers, separated by |, or "all" for all vary headers Defaults to 'all'
string $contextualPrefix The contextual prefix for contextual partials. Defaults to 'currentModule/currentAction/currentPAram1/currentvalue1' Used only by the sfViewCacheManager::remove() method
Renvoie:
string The cache key If some of the parameters contained wildcards (* or **), the generated key will also have wildcards
