Event listeners An easy to use, pluggable eventlistener architecture. Aspects such as logging, query profiling and caching can be easily implemented through the use of these listeners 2. Lazy-connecting Creating an instance of Doctrine_Connection does not connect to database. Connecting to database is only invoked when actually needed (for example when query() is being called) 3. Convenience methods Doctrine_Connection provides many convenience methods such as fetchAll(), fetchOne() etc. 4. Modular structure Higher level functionality such as schema importing, exporting, sequence handling etc. is divided into modules.
